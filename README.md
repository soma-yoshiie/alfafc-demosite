# ALFA FOOTBALL（サッカー戦術ボード／チーム運営）

少年・学校サッカー向けの「スタメン作成 ＋ 戦術アニメーション ＋ チーム運営」アプリです。
[事業計画書](../サッカーアプリ事業計画書.md) のフェーズ1コア機能（無料の集客装置となる
スタメン作成・戦術アニメ／名簿管理）を React + Next.js で実装しています。

戦術アニメーションのインタラクション（ピッチ上のなぞりでルートを記録 → タイムライン編集 →
再生）は、`football lineup/docs/index.html` のプロトタイプを参考に React へ移植しています。

## 技術スタック

- Next.js 15 (App Router) / React 19 / TypeScript
- 状態管理: React `useReducer` + Context（`components/BoardProvider.tsx`）
- 永続化: ブラウザの localStorage（`lib/storage.ts`）
- スタイル: 素の CSS（`app/globals.css`）。スマホファースト・ダーク×ライム配色。
  **PC（幅1024px以上）では2カラムのデスクトップ表示**（大きなピッチ＋右に操作パネル）に自動切替
- アイコン: `components/icons.tsx` の SVG（絵文字不使用）
- 全画面アニメ（⛶）は再生だけでなく、その場で**ルートの描画・削除・元に戻す**も可能

## 起動フロー・アカウント

アプリを開く → **スプラッシュ演出** → **ログイン** → **メニュー（ホーム）**。`components/AppFlow.tsx` が管理。
- **スタッフ**：メール＋パスワードで契約（新規登録）／ログイン。`lib/auth.ts` に端末内保存（デモ）
- **選手・保護者**：スタッフが名簿に登録した**メールアドレス**＋スタッフが設定した**共通パスワード**でログイン
  - 共通パスワードは設定画面でスタッフが設定（未設定時のデモ既定 `team2026`）
  - デモ選手：`sora@alfafc.example` ／ `team2026`
- ログイン後は**ホームメニュー**を表示（`components/HomeMenu.tsx`）。**スマホのホーム画面風のアプリアイコングリッド**（色分けタイル＋ラベル、ロック表示、挨拶・日付ヘッダー）。各機能はここから開く
  - **スタッフのみ**（選手には項目自体を出さない）：戦術ボード／**保存した戦術**／練習メニュー／**保存した練習**／チーム運営／**チャット**／**サッカーノート**／お役立ち記事／設定
  - **選手・保護者**：チーム（出欠・カレンダー・試合記録）／**チャット**／**サッカーノート**
  - 出欠：選手は**回答（○/△/×）＋理由コメントのみ**可。集計結果は閲覧不可（スタッフのみ）
  - 試合記録：**設定の「公開設定」**でスタッフが「選手に公開／非公開」を切替（非公開時は選手から閲覧不可）
  - 選手が共有戦術（チャットの添付）を開いた戦術ボードは閲覧用（保存/共有/設定など編集系はスタッフのみ表示）
- 戦術ボード等の画面の「🏠 メニューに戻る」「‹ メニュー」でホームへ。セッションは localStorage 保存
- 名簿・選手プロフィール・お役立ち記事・設定などは**全画面表示＋「‹ 戻る」**で1ステップずつ戻れる
- 「コーチ」表記は監督・トレーナー・顧問なども含むため**「スタッフ」**に統一

## 主な機能

### 1. スタメン作成（編集モード）
- 「フォーメーション変更」ボタン → 選択シートでフォーメーションを選ぶ（4-3-3 / 4-4-2 / … / 8人制プリセット）
- 空き枠タップ → 名簿から選手を配置
- トークンをドラッグで自由配置（位置からポジションを自動推定）。**別の選手に重ねてドロップすると入れ替え**（重なっていなければそのまま自由移動。近接スナップはしないので狙った位置に置ける）
- **選手をタップ → 役割メモ**：その選手の詳細な役割・指示を書ける（タップで開くメニュー内のメモ欄）。メモがある選手はトークンに 📝 を表示
- ボールも配置可能
- キャプテン設定、選手のスタメン/ベンチ入れ替え

### 2. 名簿・選手プロフィール
- メニューの「名簿」に選手管理を集約（旧チーム→選手タブを統合）
- 基本情報（名前 / 背番号 / ポジション / 身長 / 体重 / 利き足）
- **体力測定**の記録（50m走・1500m走・反復横跳び 等を自由入力で蓄積）
- **怪我履歴**（受傷日・部位・状態・メモ／スタッフ管理）
- **出席率**（チームの出欠データから自動算出・表示）
- 初回はサンプルチームを収録（編集・削除可）

### 3. 戦術アニメーション（ANIM STUDIO）
ヘッダーの ✈ アイコンで起動。
- 選手・ボールをピッチ上で**指でなぞる**とそのルートをクリップとして記録
- クリップごとに**開始タイミング・所要時間**をスライダー調整（時間差の連動が作れる）
- 再生 / 一時停止 / 先頭へ / 速度（0.5〜2x）/ スクラブ / ルート表示切替
- ⛶ 全画面再生

### 4. 保存・ライブラリ（💾／📁 アイコン）
- 名前を付けて戦術を保存。ライブラリから読込・複製・名前変更・削除
- スタッフプランは**フォルダ管理**で分類可能
- 名簿（players）はチーム共有なので、複数の戦術で**使い回し**

### 5. 共有・出力（↗ アイコン）
- **フォーメーション画像出力（PNG）**：Canvasでブランド入り画像を生成（依存ライブラリなし）
- **URL / LINE 配信**（スタッフプラン）：戦術を `#p=...` で自己完結エンコードしたリンクを生成し、
  コピー or LINE送信。受け取った側は確認のうえ読み込み（`lib/share.ts`）

### 5.5 保存・送信（戦術・トレーニングをアプリ内で配信）
戦術の保存（💾）や練習メニューの保存シートに「**送信先**（送信しない／チーム全員／個人）」を追加。
- **保存して送信する**：ライブラリ／練習メニューに保存しつつ、チャットで配信
- **保存せずに送信する**：ライブラリに残さず、その場のボード/ドリルを配信
- 送信した戦術・トレーニングは添付データを内包するため、受信側はライブラリ非依存で開ける（選手は閲覧用ボードで表示）

### 6. 練習メニュー（ドリル図・スタッフプラン）
⋯メニュー →「練習メニュー（ドリル図）」で専用エディタへ（無料プランはロック）。
- ツールパレットで **コーン／選手／相手／ボール／ゴール／マーカー** を配置（タップ）
- **動線**を指でなぞって描画：ラン（実線）・パス（破線）・ドリブル（波線）＋自動で矢印
- **動線**にラン/パス/ドリブルに加え「**ライン（直線・矢印なし）**」
- **ゴールの向き**：移動ツールでゴールをタップすると90°ずつ回転
- **選手・相手の丸サイズ**：大／中／小から選択（「丸」チップで切替）
- ピッチ表示：ハーフ／**フル縦**／**フル横（左右で攻守）**／ブランク（フルのセンターサークルは実寸寄りに縮小）
- 移動・削除・元に戻す・全消去、タイトル・メモ
- 練習メニュー単位で保存・読込、**PNG画像出力**（`lib/exportDrill.ts`）

### 6.5 チーム運営（チームプラン・⋯メニュー →「チーム（出欠・連絡）」）
別画面の TEAM HUB（`components/TeamHub.tsx` / `TeamProvider.tsx`）。チームプラン限定。
- **デモ用ロール切替**：スタッフ（管理）⇄ 選手・保護者（端末内で擬似マルチユーザー）
- **名簿・選手プロフィール**：チーム運営の「名簿」タブに集約（選手登録・プロフィール・体力測定・怪我履歴）
- **出欠管理**：練習/試合の予定を作成、選手は ○出席/△未定/×欠席 で回答、スタッフは集計と一覧編集
- **カレンダー**：**月表示**（練習=青/試合=橙の大きめドット）と**リスト表示**（日付＋横長の予定バーが縦に並ぶアジェンダ）を切替可能。日付/バータップで**予定の詳細**（日時・場所・メモ・出欠状況）を表示。スタッフはそこから**編集・削除**が可能
- **試合記録**：対戦相手・スコア・**得点者（分・アシスト）・交代**・メモを記録。一覧＝過去データ蓄積、タップで詳細
  - **大会別**：登録した**大会（カップ戦・リーグ）ごと**に分類。上部の大会チップで絞り込み、絞り込んだ範囲で勝敗・得失点・ランキングを集計。「＋ 大会を登録・管理」で大会の追加/削除、試合フォームで大会を選択（その場で新規登録も可）
- **簡易スタッツ**：勝-分-敗・得点/失点・**得点ランキング／アシストランキング**を自動集計（大会フィルタ連動）
- **公開設定**：**設定画面の「公開設定」**で試合記録を選手・保護者に「公開／非公開」で切替（非公開時、選手ビューは閲覧不可）
- **選手管理（スタッフのみ）**：「選手」タブで各選手の**出席率**と**怪我履歴**（受傷日・部位・状態・メモ）を管理
- **ベンチ→ピッチのドラッグ交代**：ベンチの選手をピッチ上の枠へドラッグ＆ドロップで投入／交代
> マルチユーザー/別端末同期・共同編集は本来サーバーが必要。本MVPは単一端末のローカルデモ。

### 6.6 チャット（独立メニュー・`components/ChatScreen.tsx`）
連絡機能をチーム運営から切り離し、**ホームの独立メニュー「チャット」**に。
- スタッフ：会話一覧（**チーム全員**＋選手ごと）→タップで全画面チャット
- 選手・保護者：自分とスタッフのDM＋チーム連絡を1つのタイムラインで表示・返信
- **戦術・トレーニング(ドリル図)・画像・動画**を添付可能（受信側はタップで戦術/トレーニングを閲覧画面に読込、画像/動画はその場で再生）。戦術/トレーニングは**保存時にも送信**できる（下記「保存・送信」）。画像は自動縮小、動画は約3.5MBまで（端末内デモ）

### 6.7 サッカーノート（独立メニュー・`components/NotebookScreen.tsx`）
**選手が提出する振り返りノート**。ホームの独立メニュー「サッカーノート」。**3バケツ（試合 / 練習 / 自主練）のタブ構成**。
- 共通：日付・**コンディション（5段階の絵文字）**・自由メモ。選手は記入→提出、自分のノートの閲覧・編集・削除。スタッフは全選手分を閲覧し詳細で**フィードバックコメント**を送信
- **試合ノート**：
  - ① **スタメン・フォーメーション記録**（前半/後半/延長のフェーズ別。自チーム陣形＋**自分のポジションをミニコートでタップ選択**＋相手陣形）。一覧に「今シーズンの出場ポジション（例：右SB◯試合）」を自動集計
  - ② **自分のプレーエリア記録**（コートをタップして 受けた/シュート/ミス を記録）＋**AI分析**（左右・中央・敵陣進入の割合と要約）
- **練習ノート**：
  - ⑥ **練習前の目標＋達成度（%）**、④ **気づき（複数）**、⑪ **チーム共有**（ON にすると他選手の「チームの共有」タブに表示）
- **自主練ノート**：⑩ リフティング/ランニング/筋トレ等の種目ログ＋🔥**連続記録日数（ストリーク）**
- **AI**：`lib/ai.ts` の `AiClient` インターフェースで抽象化。現在は**ローカル決定論的要約**（API不要）。将来 Claude API（claude-opus-4-8 / sonnet）実装へ `aiClient` を差し替えるだけで切替可能
- **コーチからの配信（`components/DeliverViews.tsx`／キー `soccer_tactics_coachdeliver_v1`）** ＝ノート内に統合：
  - ③ **練習メニュー配信**（区分・内容）→ 選手は理解度／難易度（★5段階）＋感想で回答（練習タブ「📣コーチから」）
  - ⑤ **個人課題**（全員/個人指定）→ 選手は 実践できた／一部できた／できなかった で回答（練習タブ）
  - ⑨ **ポジション理解度テスト**（選択式・複数設問）→ 選手は回答＝即採点、コーチは**設問別正答率＋個人スコアでチーム理解度を可視化**（練習タブ）
  - ⑦ **試合前ミーティング**（攻撃/守備のポイント）→ 選手は「確認」ボタン＋試合後に「実行できたか」を記録（試合タブ「📣コーチから」）
- **試合ノートの追加項目**：⑧ **ベストプレー／反省プレー＋動画URL**、⑫ **AI試合レポート**（出場時間・ポジション[陣形から自動]・良かった点・改善点 → AI要約）
- 保存は専用キー `soccer_tactics_notebook_v1`（BoardProvider 管理／旧フラット形式は読込時に練習ノートへ自動移行）。ミニコートは `components/MiniPitch.tsx`（BoardProvider 非依存の表示専用）。AIは全て `lib/ai.ts` の `AiClient`（`analyzePlayArea`／`summarizeMatchReport`）でローカル生成、将来Claude API差し替え可能
- ⑪ チーム共有は練習ノートの「チームに共有」トグル＋「チームの共有」スコープ（独立タブにしない）

### 6.8 横断検索＆コーチ・ダッシュボード（`components/NotebookTools.tsx`）
サッカーノート画面ヘッダーの 🔍／📊 から開く（📊はコーチのみ）。**いずれもルールベースのみ・AI/API不要（将来のAI差し替えは `lib/ai.ts` / `lib/coaching.ts` の関数単位で可能）**。
- **🔍 横断検索**：全ノートを横断し、キーワード（気づき・目標・メモ・相手名・AI要約 等）＋種別（試合/練習/自主練）で絞り込み。コーチは選手でも絞り込み可。シーズン後の振り返りに対応
- **📊 コーチ・ダッシュボード**（`lib/coaching.ts`）：チームサマリ（平均出席率・今週のノート数・未コメント数・自主練継続人数）＋**選手ごとのKPIカード**（出席率・ノート数・自主練/ストリーク・課題達成・テスト正答率・コンディション・怪我）。`generatePlayerAlerts()` が**ルールベースの自動コメント**を生成（例：「ノート提出が14日途絶えています」「出席率が60%に低下」「自主練7日連続」「未コメント◯件」「テスト正答率が低い」）。警告の多い選手を上位に表示。カードのタップでその選手の横断検索へ
- AIロードマップ：**フェーズ1＝本ルールベース** → フェーズ2＝Claude API（`aiClient`/`generatePlayerAlerts` を差し替え）→ フェーズ3＝自社ローカルLLM

### 6.9 通知＆シーズンレポート（`lib/notifications.ts` / `lib/seasonReport.ts`）
**いずれもルールベースのみ・APIなし**。
- **🔔 通知**：ノート画面ヘッダーの🔔とホームの「サッカーノート」タイルに**未読バッジ**。`buildEventNotifications`（実tsのイベント＝未読判定：選手のノート提出/配信回答[コーチ]、新着配信/コメント[選手]）＋`buildDigest`（しきい値の「今日のまとめ」）。既読は `soccer_tactics_notif_seen_v1`（identityごとのts）。各通知タップで該当ノート/配信/選手へジャンプ。※端末内デモのためアプリ起動中のみ（実プッシュ/LINE/メールはフェーズ2のサーバ実装）
- **📄 シーズンレポート**：ダッシュボードの選手カード「📄レポート」／選手のノート画面ヘッダー📄から。期間（開始〜終了）で集計し、出場・出席率・ノート数・自主練/最長ストリーク・課題達成・テスト平均・出場ポジション・代表的な気づき・ベストプレー・コーチコメント＋**ルールベース総括文**（`generateSeasonSummary`）を表示。**白い紙レイアウトで「🖨 印刷/PDF保存」（`window.print()`＋印刷用CSS）と「🖼 画像で保存」（Canvas PNG＝`renderSeasonImage`）の両対応**

### プラン（クラブ単位の有料サブスク・B2B／`lib/types.ts` の `PLAN_INFO`）
**無料プランなし**。**安全設計・コア機能・差別化機能（出席率の可視化・サッカーノート・戦術配信）は全プラン共通＝最安から使える**。プラン差は「**規模**」（チーム数・選手数・コーチ席数）のみ。**データは全プラン永続保存**。上限超過時は上位プランへ移行（個別オプションなし＝バンド制）。
